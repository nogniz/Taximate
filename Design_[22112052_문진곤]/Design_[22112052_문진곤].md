## 1. Introduction

### 1.1 Summary
현대 사회의 대학생들에게 이동 수단은 단순한 이동을 넘어 시간과 비용의 효율성을 결정짓는 중요한 요소이다. 최근 다양한 모빌리티 서비스가 등장했으나, 대학생들이 매일 부담하기에는 택시 비용이 너무 높고, 기존의 카풀 방식은 신원 확인의 불확실성과 실시간 매칭의 어려움으로 인해 직관적인 편리함을 제공하지 못하고 있다. 이에 대학생들이 겪는 경제적 부담을 덜어주고, 안전하면서도 직관적인 동승 환경을 제공하기 위해 기획된 플랫폼이 바로 “Taxi Mate”이다.

### 1.2 Important Points of Design
* **MVC 기반 아키텍처 설계**: UI(View) 영역을 담당하는 클라이언트단과 핵심 비즈니스 로직 및 멀티스레딩을 처리하는 백엔드 서버(Core) 영역을 철저히 분리하여 결합도를 낮추고 유지보수성을 극대화한다.
* **실시간 통신 최적화**: WebSocket(소켓 통신) 및 비동기 멀티프로세싱 스케줄러를 도입하여 0.5초 미만의 실시간 채팅 및 매칭 엔진 연동을 보장한다.
* **외부 서비스의 모듈화**: Google Maps API(위치 검증) 및 외부 PG사(결제 대행) 시스템과의 통신 체계를 독립적인 컨트롤러 클래스로 격리하여 외부 환경 변화에 유연하게 대응하도록 설계한다.

## 2. Class diagram

본 프로젝트는 시스템의 정적 구조를 명확히 하고 레이어 간 데이터 결합도를 낮추기 위해, 사용자가 상호작용하는 클라이언트단(View)과 핵심 비즈니스 로직을 처리하는 백엔드 플랫폼단(Core)을 철저히 분리하여 설계하였다.

### 2.1 Class Diagram Specifications (Detailed Tables)

#### 1) UserController
* **Description**: 사용자 정보 및 재학생 인증 상태, 신뢰도 지표를 관리하는 클래스이다.

| 구분 | Name | Type | Scope | Visibility | Description |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Attributes** | student_id | str | instance | private | 사용자의 고유 학번 데이터 |
| | name | str | instance | private | 사용자의 본명 |
| | is_auth | bool | instance | private | 학교 포털 이메일 인증 완료 여부 |
| | manner_temp | float | instance | private | 상호 평가 시스템 기반 매너 온도 수치 |
| **Operations** | update_manner() | score: float | None | public | 동승 완료 후 피드백 반영 매너 온도 갱신 |
| | get_profile() | None | dict | public | 사용자 이용 통계 및 프로필 데이터 반환 |

#### 2) TaxiRoomManager (Core)
* **Description**: 동승을 위해 생성된 방(Room)들의 전체 목록과 상태를 제어하고 관리하는 클래스이다.

| 구분 | Name | Type | Scope | Visibility | Description |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Attributes** | active_rooms | list\<Room\> | instance | private | 현재 시스템에서 활성화된 동승 방 리스트 |
| | room_count | int | instance | private | 대기 및 활성화 상태인 방의 총 개수 |
| **Operations** | create_room() | info: dict | Room | public | 새로운 동승 목적지 요청 방을 생성하고 리스트에 추가 |
| | delete_room() | room_id: int | bool | public | 방 해체 또는 대기 시간 만료 시 해당 방 객체 소멸 |

#### 3) MatchEngine (Core)
* **Description**: 위치 정보(GPS)를 기반으로 목적지 좌표를 분석하여 경로 유사도를 계산하고 최적의 동승자를 찾아주는 클래스이다.

| 구분 | Name | Type | Scope | Visibility | Description |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Attributes** | min_similarity | float | static | private | 매칭 승인을 위한 최소 경로 유사도 가이드라인 (80%) |
| **Operations** | calculate_route()| r1: Route, r2: Route | float | public | 두 사용자 간의 출발지/목적지 좌표 유사도 연산 |
| | find_partners() | user_id: str | list\<User\>| public | 조건에 부합하는 가용 동승 후보군 목록 추출 및 알림 트리거 |

#### 4) RoomController
* **Description**: 출발지, 목적지, 출발 희망 시간 등 개별 동승 방의 세부 정보와 참여 인원 상태를 제어하는 클래스이다.

#### 5) MapController
* **Description**: 지도 서비스 연동 및 실시간 GPS 좌표를 수집하여 예상 거리와 예상 택시 요금 정보를 제공하는 클래스이다.

#### 6) ChatManager
* **Description**: 매칭이 완료된 사용자들 간의 소켓(Socket) 통신을 활성화하고 실시간 메시지 송수신을 관리하는 클래스이다.

#### 7) AuthManager
* **Description**: 학교 포털 이메일을 통한 신분 인증 및 인증 코드 발송/유효성 검증을 수행하는 클래스이다.

#### 8) PaymentController
* **Description**: 서비스 이용 후 최종 택시 요금을 인원수별로 계산하여 개별 정산을 산출하고 외부 PG 시스템과 연동하는 클래스이다.

#### 9) NotificationManager
* **Description**: 매칭 성공 제안, 신규 채팅 메시지 도착 등 핵심 이벤트 발생 시 실시간 푸시 알림 발송을 제어하는 클래스이다.

#### 10) UIManager
* **Description**: 지도 위의 핀 이미지, 매칭 대기 팝업, 채팅 인터페이스 등 클라이언트 앱의 시각적 컴포넌트를 갱신하는 클래스이다.

## 3. Sequence diagram

### 3.1 Real-Time Match & Join Room (실시간 동승 매칭 및 방 참여 과정)
사용자가 목적지를 설정하여 방을 개설하거나 최적의 동승 파트너를 탐색하여 방에 참여할 때의 시간적 객체 간 상호작용 흐름이다.

![sequence_diagram](./sequence_diagram.png)
1. **상호작용 순서**:
    * 사용자가 클라이언트 앱에서 목적지 검색 후 동승 요청을 등록한다.
    * `MapController`가 즉시 사용자의 GPS 좌표를 수집하고 `MatchEngine`으로 연산을 요청한다.
    * `MatchEngine`은 데이터베이스 내 활성화된 `TaxiRoomManager`의 방 목록을 스캔하여 경로 유사도를 계산한다.
    * 경로 유사도가 80% 이상인 최적의 방이 탐색되면, `NotificationManager`가 대기 중인 상대 사용자에게 매칭 제안 알림을 발송한다.
    * 사용자가 알림을 확인하고 참여(Join)를 수락하면 `RoomController`의 참여자 상태가 `Waiting`에서 `Active`로 전이되며 실시간 채팅 채널이 즉시 개설된다.

## 4. State machine diagram

### 4.1 Client-Side App Lifecycle State Machine Diagram
본 다이어그램은 사용자가 애플리케이션을 구동하여 최종 결제 및 정산을 완료할 때까지 객체가 가지는 수명 주기와 상태 전이 과정을 정의한다.
![state_machine_diagram](./state_machine_diagram.png)
* **OFF / Idle**: 애플리케이션이 실행되기 전이거나 백그라운드에서 대기 중인 상태.
* **Authentication**: 앱 실행 후 최초 1회 진행하는 학교 이메일 포털 인증 상태.
* **Main Map (Searching)**: 인증 완료 후 실시간 GPS를 기반으로 주변의 동승 방을 탐색하거나 생성을 대기하는 상태.
* **Chatting (Active)**: 매칭 엔진에 의해 동승 매칭이 성사되어 전용 소켓 채널에서 픽업 지점을 조율하는 상태.
* **Driving**: 모든 인원이 링크되어 목적지로 택시를 타고 이동 중인 운행 상태.
* **Settlement (Billing)**: 목적지 하차 후 최종 요금을 입력받아 n분의 1 자동 결제를 진행하고 상호 신뢰도 평가를 마친 뒤 다시 Main Map으로 복귀하는 상태.

## 5. Implementation requirements

### 5.1 H/W platform requirements
* **Client Device**: Android 10.0 이상 또는 iOS 15.0 이상의 스마트폰 기기 (GPS 수신 모듈 탑재 필수).
* **Cloud Server (AWS EC2 / 로컬 서버)**:
    * **Processor**: Intel or AMD 8-core or higher CPU (비동기 소켓 멀티스레딩 처리를 위함).
    * **Memory**: 8GB RAM 이상.
    * **Storage**: 20GB 이상의 사용 가능한 SSD 저장 공간.

### 5.2 S/W platform requirements
* **Operating System**: Ubuntu 22.04 LTS (Production Server 환경).
* **Implementation Language / Environment**: Java 17 (Spring Boot Framework), WebSocket API.
* **Database**: MySQL 8.0 (영속성 데이터 유저 로그 저장), Redis (실시간 세션 및 대기 매칭 큐 관리).

## 6. Glossary

용어사전에 대해선 다음의 표와 같다.

| Terms | Description |
| :--- | :--- |
| **패널티 (Penalty)** | 사용자가 동승 방에 참여한 후 정당한 사유 없이 픽업 시간 5분 전까지 나타나지 않거나, 반복적으로 매칭을 취소할 경우 부여되는 불이익이다. 패널티 적립 시 일정 기간 서비스 이용이 제한되며, 매너 온도가 하락하게 된다. |
| **매너 온도** | 사용자의 서비스 이용 태도와 신뢰도를 시각화한 지표이다. 기본 36.5도에서 시작하며 동승 완료 후 상호 평가를 통해 온도가 오르거나 내린다. 온도가 높을수록 우선 매칭권이 부여되며, 일정 온도 이하의 사용자는 이용이 금지된다. |
| **Home** | 시스템에 처음 접속 하였을 때 볼 수 있는 화면이다. 현재 내 주변의 실시간 동승 대기 방(Room) 목록과 사용자의 현재 위치(My Location)를 지도 기반으로 직관적으로 확인할 수 있다. |
| **Attribute** | 객체지향언어인 JAVA로 예를 들면 멤버 필드라고 할 수 있다. Taxi Mate 시스템에서는 사용자 정보(학번, 이름), 방 상태(Active/Waiting/Full), 결제 정보 등이 각 클래스의 속성(Attribute)에 해당한다. |
| **Operation** | 객체지향언어인 JAVA로 예를 들면 메서드라고 할 수 있다. 시스템 내에서 '동승 방 생성(Create Room)', '채팅 메시지 전송(Send Message)', '위치 검증(Verify Location)' 등 클래스가 수행하는 구체적인 기능을 말한다. |
| **Notice** | 관리자가 시스템 전체 공지사항이나 안전 수칙을 사용자에게 전달하는 기능이다. 새로운 매칭 제안이나 채팅 메시지 도착 등 사용자에게 발생하는 각종 실시간 알림을 포함한다. |
| **GUI** | 그래픽 사용자 접속기(Graphical User Interface). 지도 상의 핀 이미지, 현재 매칭 상태 바, 채팅 화면 등 사용자가 눈으로 보고 직관적으로 조작할 수 있도록 돕는 시각적 인터페이스를 말한다. |
| **경로 유사도** | 두 사용자 간의 이동 경로가 얼마나 일치하는지를 나타내는 수치이다. MatchEngine 클래스가 지도 API를 통해 수집된 목적지 좌표를 분석하여 계산하며, 매칭 여부를 결정하는 핵심 기준이 된다. |
| **WebSocket** | 동승자 간 실시간 채팅 기능을 구현하기 위해 사용되는 통신 규약이다. 서버와 클라이언트 사이에 연결을 유지하여 메시지 전송 시 지연 시간을 0.5초 이내로 단축시켜 원활한 소통을 지원한다. |
| **PG (Payment Gateway)** | 온라인 결제 및 송금을 대행해 주는 외부 서비스이다. 택시 이용 후 산출된 요금을 인원수대로 정산하여 안전하게 결제 처리를 수행하고 그 결과를 시스템에 반환한다. |
