## 1. Introduction

### 1.1 Summary
현대 사회의 대학생들에게 이동 수단은 단순한 이동을 넘어 시간과 비용의 효율성을 결정짓는 중요한 요소이다. 최근 다양한 모빌리티 서비스가 등장했으나, 대학생들이 매일 부담하기에는 택시 비용이 너무 높고, 기존의 카풀 방식은 신원 확인의 불확실성과 실시간 매칭의 어려움으로 인해 직관적인 편리함을 제공하지 못하고 있다. 이에 대학생들이 겪는 경제적 부담을 덜어주고, 안전하면서도 직관적인 동승 환경을 제공하기 위해 기획된 플랫폼이 바로 "Taxi Mate"이다.

본 버전(v2.1)은 Spring Boot 3.1.5 + Java 17 기반으로 실제 구현된 백엔드 서버를 기준으로 작성되었으며, 이메일 인증 → 회원가입 → JWT 인증 → 방 개설 → 자동 매칭 → 탑승완료/출발/강퇴 → WebSocket 알림 → 순차하차 정산 → 이용내역 DM까지의 전체 플로우가 구현 및 테스트 완료되었다.

### 1.2 Important Points of Design
* **Spring Boot MVC 아키텍처**: Controller → Service → Repository 3계층 구조로 레이어 간 결합도를 낮추고 유지보수성을 극대화하였다.
* **JWT 기반 Stateless 인증**: Spring Security 6.1.5와 JJWT 0.11.5를 사용하여 세션 없이 토큰 기반 인증을 구현하였다. JwtAuthFilter가 모든 보호 엔드포인트에서 Bearer 토큰을 검증한다.
* **실시간 WebSocket 알림**: STOMP over SockJS 방식으로 WebSocket을 구현하였으며, NotificationService가 `/sub/notification/{roomId}` 토픽으로 MATCHED, FULL, BOARDED, KICKED, COMPLETED, CANCELLED, PAYMENT, PAYMENT_DONE 이벤트를 푸시한다.
* **Haversine 기반 자동 매칭**: MatchEngine이 두 지점의 위도/경도로 Haversine 공식을 적용해 직선거리를 계산하고, 임계값(2km) 이내 + 출발 시각 ±30분 이내인 경우 자동 합류(MATCHED) 처리한다.
* **26개 사전 정의 장소 선택 피커**: 출발지/목적지를 DAEGU_COORDS 맵과 1:1 대응되는 선택 목록에서 지정하여, 자유 입력으로 인한 좌표 오류(예: 경대병원역 vs 칠곡경대병원역)를 근본적으로 제거하였다.
* **탑승 흐름 제어 (boardedUsers)**: 각 참여자가 '탑승완료' 처리 후 방장이 `isAllBoarded() = true`를 확인해야만 출발 처리가 가능하다. 방장은 강퇴(kickUser) 권한도 가진다.
* **순차하차 정산 (Sequential Drop-off)**: PaymentService가 각 참여자의 목적지까지 거리를 기준으로 가까운 순서대로 정렬하고, 구간별 요금을 해당 구간 탑승자 수로 균등 분담한다.
* **이메일 인증 (Brevo Transactional API)**: Railway 환경에서 SMTP 포트(25/465/587)가 차단되어 JavaMailSender 대신 Brevo REST API(HTTPS)를 사용한다. `BREVO_API_KEY` 미설정 시 로컬 폴백으로 고정 코드 `123456`을 사용한다.
* **방 상태 머신 (5단계)**: TaxiRoom의 상태는 WAITING → FULL → COMPLETED → SETTLED / CANCELLED로 엄격하게 제어된다.

## 3. Sequence diagram

### 3.1 Full E2E Flow (STEP 1~14)
사용자가 이메일 인증으로 회원가입 후 로그인하여 동승 방을 개설하고, 자동 매칭 → 탑승완료 → 출발 → 정산 → 동승자 평가 → 1:1 DM까지의 전체 상호작용 흐름.

![sequence_diagram](./sequence_diagram.png)

1. **STEP 1~2 — 이메일 인증**: `POST /api/auth/email`로 Brevo API(HTTPS REST) 발송. `POST /api/auth/verify`로 3분 내 검증. 로컬 환경(`BREVO_API_KEY` 미설정) 시 고정 코드 `123456` 사용.
2. **STEP 3~4 — 회원가입 & 로그인**: `POST /api/users/register` 등록. `POST /api/users/login`으로 24시간 JWT 발급.
3. **STEP 5 — 프로필 조회**: `GET /api/users/me`로 학번, 이름, 매너온도 반환.
4. **STEP 6 — 방 개설**: `POST /api/rooms`로 WAITING 상태 방 생성.
5. **STEP 7 — 자동 매칭**: `POST /api/rooms/match`. MatchEngine이 Haversine ≤2km + ±30분 이내 방에 자동 합류. MATCHED/FULL 알림 발송.
6. **STEP 8 — 탑승완료**: 각 참여자가 `POST /api/rooms/{id}/board`. boardedUsers에 등록, BOARDED(n/total) 알림.
7. **STEP 9 — 출발**: 방장이 `PATCH /api/rooms/{id}/depart`. isAllBoarded() 검증 후 COMPLETED 전환.
8. **STEP 10 — 강퇴**: 방장이 `DELETE /api/rooms/{id}/kick/{uid}`. 참여자 제거 및 KICKED 알림.
9. **STEP 11~12 — 정산**: `POST /api/payments/{id}`로 순차하차 정산. `PATCH /api/payments/{id}/pay`로 납부 처리. 전원 납부 완료 시 SETTLED + PAYMENT_DONE 알림.
10. **STEP 13 — 평가**: `POST /api/users/{uid}/rate`로 매너 온도 갱신(+0.5/-0.3°C).
11. **STEP 14 — 1:1 DM**: `GET /api/chat/dm-id/{uid}`로 채널 확인 후 WS `/sub/chat/dm/{dmId}` 연결.

## 4. State machine diagram

### 4.1 TaxiRoom State Machine Diagram (v2.0)

![state_machine_diagram](./state_machine_diagram.png)

| State | Description | 진입 조건 | 가능한 전이 |
| :--- | :--- | :--- | :--- |
| **WAITING** | 방 개설 직후 참여자 대기 중 | 방 생성 시 초기 상태 | → FULL, → COMPLETED, → CANCELLED |
| **FULL** | 최대 인원(maxParticipants) 충족 | 참여자 수 == maxParticipants | → WAITING(강퇴), → COMPLETED, → CANCELLED |
| **COMPLETED** | 방장이 출발 처리 완료 | 방장 `/depart` 호출 + isAllBoarded=true | → SETTLED |
| **SETTLED** | 전원 정산 납부 완료 | 모든 paidStatus = true | Terminal |
| **CANCELLED** | 방장이 방 취소 | 방장 `/cancel` 호출 | Terminal |

**boardedUsers 흐름**: 각 참여자가 `POST /board` → boardedUsers.add(userId) → BOARDED(n/total) 알림. 방장이 `PATCH /depart` 호출 시 `isAllBoarded()` 검증 → false이면 400 오류.

**WebSocket 알림 목록**:

| 이벤트 | 발생 시점 |
| :--- | :--- |
| MATCHED | 자동 매칭 또는 수동 참여 성공 |
| FULL | 최대 인원 충족 |
| BOARDED | 참여자 탑승완료 처리 |
| KICKED | 방장 강퇴 처리 |
| COMPLETED | 방장 출발 처리 |
| CANCELLED | 방장 방 취소 |
| PAYMENT | 정산 시작 (fareSplit 포함) |
| PAYMENT_DONE | 전원 납부 완료 |

## 5. Implementation requirements

### 5.1 H/W platform requirements
* **Cloud Server (개발/테스트 환경)**:
    * **Processor**: Intel or AMD 멀티코어 CPU.
    * **Memory**: 4GB RAM 이상 (H2 In-Memory DB 사용으로 최소화).
    * **Storage**: 5GB 이상의 여유 공간.

### 5.2 S/W platform requirements
* **Operating System**: Windows 10/11, macOS, Ubuntu 22.04 LTS 모두 지원.
* **Implementation Language / Environment**:
    * Java 17, Spring Boot 3.1.5
    * Spring Security 6.1.5 (JWT Stateless 인증)
    * Spring WebSocket + STOMP + SockJS (실시간 알림 + 채팅)
    * JJWT 0.11.5 (JSON Web Token)
    * Lombok, Spring Data JPA
* **Database**: H2 In-Memory Database (개발/테스트용)
* **Build Tool**: Gradle
* **이메일**: Brevo Transactional Email API (HTTPS REST, `https://api.brevo.com/v3/smtp/email`). Railway SMTP 포트(25/465/587) 차단 우회. 로컬 환경(`BREVO_API_KEY` 미설정) 시 고정 코드 `123456` 폴백.
* **좌표 조회**: 26개 사전 정의 DAEGU_COORDS 맵 (외부 Map API 불필요)
* **프론트엔드**: 단일 HTML SPA (`index.html`), SockJS + STOMP.js
* **API 테스트**: IntelliJ HTTP Client (`test.http`, STEP 1~14)

### 5.3 Security Requirements
* 모든 보호 엔드포인트는 `Authorization: Bearer <JWT>` 헤더 필요.
* JWT payload에 studentId(sub), iat, exp(24h) 포함.
* 비밀번호는 BCryptPasswordEncoder로 단방향 해시 저장.
* 방장 전용 작업(출발/강퇴/취소)은 hostId 검증 후 처리.
* CORS는 전체 허용(`@CrossOrigin(origins = "*")`), 운영 환경에서는 도메인 제한 필요.

## 6. Glossary

| Terms | Description |
| :--- | :--- |
| **JWT** | 서버 세션 없이 인증 상태를 유지하기 위한 자가 검증 토큰. Header.Payload.Signature 구조. studentId를 sub 클레임에 담아 발급하며, 24시간 유효. |
| **STOMP** | Simple Text Oriented Messaging Protocol. WebSocket 위에서 pub/sub 메시지 라우팅 서브프로토콜. |
| **SockJS** | WebSocket 미지원 환경에서 HTTP Long-polling 등의 폴백 전송을 제공하는 라이브러리. |
| **Haversine 공식** | 지구 표면 위 두 위도/경도 좌표 간의 최단 직선거리 계산 공식. 임계값 2km + ±30분 이내. |
| **순차하차 정산** | 총 택시비를 구간(leg)별로 나누어 해당 구간 탑승 인원 수로 균등 분담. 멀리 가는 사람이 더 많이 부담. |
| **boardedUsers** | 각 참여자의 탑승완료 여부를 추적하는 Set. isAllBoarded() = true 시에만 방장의 출발 처리 가능. |
| **DAEGU_COORDS** | MatchEngine의 26개 대구 주요 장소 하드코딩 좌표 맵. 외부 Map API 없이 장소명-좌표 변환 수행. |
| **매너 온도** | 사용자 신뢰도 지표. 기본 36.5°C. 👍 +0.5°C / 👎 -0.3°C. 0~100°C 범위. |
| **Brevo Transactional Email API** | HTTPS REST 방식의 이메일 발송 서비스. Railway SMTP 포트 차단 우회. 임의의 `@yu.ac.kr` 수신자에게 발송 가능. |
| **로컬 폴백 코드** | `BREVO_API_KEY` 환경변수가 `placeholder`일 때 이메일 발송 없이 인증코드 `123456`을 고정 사용하는 개발 편의 기능. |
| **WAITING** | 방 개설 직후 참여자 모집 중인 상태. |
| **FULL** | 최대 인원 충족 상태. |
| **COMPLETED** | 방장 출발 처리 후 정산 진입 가능 상태. |
| **SETTLED** | 전원 정산 납부 완료 상태. Terminal. |
| **CANCELLED** | 방장 취소 상태. Terminal. |
| **H2 In-Memory DB** | 서버 프로세스 메모리 내에서만 동작하는 내장형 RDBMS. 서버 재시작 시 초기화. |
| **BCrypt** | 비밀번호 단방향 해시 알고리즘. Salt 포함으로 동일 비밀번호도 매번 다른 해시 생성. |
